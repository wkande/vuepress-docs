# AWS Cloud Credentials

    Airnode will deploy serverless functions to AWS Lambda. Follow these steps to create an IAM User with programmatic access and the necessary permissions.

    1. Login (create an account if needed) to the AWS Console (opens new window).

    2. From the AWS Management Console page access: All Services > Security, Identity, & Complicance > IAM.

    3. From the Identity and Access Management (IAM) page select Users from the left SideBar.

    4. On the User List page select the Add user button.

    5. On the Add Users page enter a User name. Select the check box for Programmatic access. Select the Next: Permissions button.

    6. Set permissions for the new User. For this tutorial select Lambda Full Access. Be sure to set the proper permissions for any production environment. Select the Next: Tags button.

    7. Tags are optional. Select the Next: Review button.

    8. Verify AWS access type is "Programmatic access - with an access key" and Managed Policy is "AWSLambda_Full_Access". Select the Next: Create User button.

    9. The final page shows the newly created user and its keys. Be sure to write down the Secret access key. It can only be viewed once and then is hidden by AWS.

    The keys (Access key id and Secret access key) are used by Airnode to install serverless functions to AWS. See the tutorial airnode-starter to learn more.
